
# Architecture Decision Records (ADR)

---

## ADR-007: 좌측 사이드바 네비게이션 도입 (Sidebar Navigation Architecture)

- **날짜:** 2024-08-05
- **상태:** 채택됨

### 컨텍스트 (Context)
- 기존 앱은 모든 네비게이션 버튼이 상단 `sticky` 영역에 밀집되어 있었습니다.
- 콘텐츠 종류(대회, 데이터, 코드, 강좌 등)가 늘어남에 따라 상단 탭 방식은 시각적으로 복잡해지고, 작은 화면에서 버튼이 줄바꿈되는 등 레이아웃 파손 문제가 발생했습니다.
- 사용자가 "대회"와 "학습"이라는 두 개의 큰 맥락 사이를 더 명확하게 인지하고 전환할 수 있는 구조가 필요했습니다.

### 결정 (Decision)
- 앱의 기본 레이아웃을 **2-Column (Sidebar + Main Content)** 구조로 변경하기로 결정했습니다.

### 근거 (Rationale)
- **인지적 분류:** 메뉴를 좌측에 배치함으로써 "시스템의 구조"와 "현재의 콘텐츠"를 시각적으로 분리하여 사용자 경험을 개선합니다.
- **수직 공간 확보:** 상단의 두터운 메뉴 레이어를 제거하여 메인 콘텐츠 리스트를 더 많이 보여줄 수 있습니다.
- **확장성:** 향후 새로운 콘텐츠 카테고리가 추가되어도 사이드바 리스트에 항목을 추가하는 것만으로 대응이 가능하여 디자인 일관성을 유지하기 쉽습니다.
- **전문성:** 대시보드 형태의 서비스에서 사이드바는 표준적인 UI 패턴으로, 사용자에게 더 전문적이고 신뢰감 있는 인상을 줍니다.

### 고려된 대안 (Alternatives Considered)
- **Multi-level Top Nav:** 드롭다운 방식의 상단 메뉴는 접근성이 떨어지고 모바일에서 구현이 복잡합니다.
- **Bottom Navigation (Mobile Only):** 모바일에서는 유용하지만 데스크톱 대화면에서는 공간 낭비가 심하므로 채택하지 않았습니다.

### 결과 (Consequences)
- `App.tsx`의 레이아웃 코드가 복잡해졌으나, `Sidebar` 컴포넌트의 독립성을 확보하여 관리 효율성이 높아졌습니다.
- 테마 시스템이 사이드바까지 확장되어 디자인 일관성이 강화되었습니다.
